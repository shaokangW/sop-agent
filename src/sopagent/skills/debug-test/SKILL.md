# 调试失败测试

## 工作流
1. **复现**:用 `bash` 跑测试命令(优先 `python -m pytest -q <测试文件> -x`),拿到完整报错
2. **读报错**:从报错提取:错误类型、文件:行号、断言期望 vs 实际;若是多行 traceback,定位最内层用户代码帧
3. **读上下文**:用 `read_file` 读出错文件对应行号附近 ±20 行(带 `offset`),理解意图
4. **定位根因**:区分「测试本身错」vs「被测代码错」——不要为了通过测试而改测试断言,除非测试本身有笔误
5. **最小改动**:用 `edit_file` 改一处,不要顺手重构;改动要能解释清楚为什么能修
6. **重跑验证**:只跑受影响的测试(不要每次全量),通过后再跑相关模块
7. 完成前 `bash` 跑一遍确认绿

## 注意
- 危险操作(edit_file/bash 写)会触发用户审批,被拒就换方案
- 改不动就退一步重新读上下文,不要靠猜反复试
- 引用 `refs/checklist.md`(如有)核对常见坑
